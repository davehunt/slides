Why test automation for Firefox OS?

2.5 million lines of code. 30 commits per day. 255 contributors...
and that's just the user interface!

We also have WebAPI testing, application performance testing, update testing, and much more...

The only way to keep up is to automate!

Buildbot and Tinderboxpushlog

  • Continuous integration with ~65.000 tests based on check-in
  • Various testsuites: XPCShell (X), Mochitests (M), Reftests (R), Marionette (Mn), Gaia UI (G)
  • Different types of devices (Panda, Unagi, Otoro, and emulator)

How Gaia UI tests work

  • Executed on three Unagi handsets via Jenkins CI
  • Uses the Marionette test framework
  • Python tests which include JavaScript code
  • Easy to install: pip install gaiatest
  • Cleans up between tests
  • Takes screenshots on failure

Repository: https://github.com/mozilla/gaia-ui-tests

Handsets need love too!

Gaia UI continuous integration

  • Gecko is downloaded whenever a new build is available (daily)
  • Jenkins CI polls the Gaia repository every 15 minutes:
    • Handset is flashed with latest Gecko
    • Handset is flashed with latest Gaia
    • Tests are run on the handset
  • In addition to the Jenkins dashboard, results are reported to:
    • Mailing list: webqa-ci@mozilla.com
    • IRC: #mozwebqa
    • Autolog
